# Phase 2 ‚Äî Plan 02: Frontend Listings UI

## Goal
Build the Find Homes search page, property detail page (all 3 categories), dashboard listing management, and file upload flow.

---

## Tasks

### 1. Find Homes page ‚Äî `frontend/src/app/find-homes/page.tsx`
- Server component ‚Äî fetch listings via `serverFetch("/api/listings?...")`
- Pass URL search params as filters to API
- Render `FilterSidebar` + `SearchListingCard` grid
- Pagination controls

### 2. Filter sidebar ‚Äî `frontend/src/components/find-homes/FilterSidebar.tsx`
- Client component ‚Äî manages filter state
- Controls: category tabs, price range slider, bedrooms/bathrooms, property type, city, sort order
- On change: updates URL search params (router.push)

### 3. Search listing card ‚Äî `frontend/src/components/find-homes/SearchListingCard.tsx`
- Card view: image, title, price, location, bed/bath/sqm badges, verified badge, favorite button
- Clicking navigates to `/property/[id]`

### 4. Search listing row ‚Äî `frontend/src/components/find-homes/SearchListingRow.tsx`
- Row/list view variant of the listing card (toggle between grid/row in Find Homes)

### 5. Property detail page ‚Äî `frontend/src/app/property/[id]/page.tsx`
- Server component ‚Äî fetch `GET /api/listings/{id}` via `serverFetch`
- If `category === "shared_housing"` ‚Üí render shared housing layout
- Otherwise ‚Üí render standard property layout
- ISR revalidate: 60s

**Standard layout components:**
- `frontend/src/components/property/PropertyHero.tsx` ‚Äî image gallery, price, key stats
- `frontend/src/components/property/PropertyInfo.tsx` ‚Äî description, amenities, map (Leaflet)

**Shared housing layout components:**
- `frontend/src/components/shared-housing/CompatibilityScore.tsx` ‚Äî compatibility score display + apply button
- `frontend/src/components/shared-housing/SharedAmenities.tsx` ‚Äî shared amenities list

### 6. Shared housing redirect ‚Äî `frontend/src/app/shared-housing/[id]/page.tsx`
- Server component ‚Äî `redirect("/property/" + params.id)`

### 7. Dashboard: add listing modal ‚Äî `frontend/src/components/dashboard/AddListingModal.tsx`
- Client component ‚Äî multi-step form (basic info ‚Üí details ‚Üí media ‚Üí submit)
- File upload: call `POST /api/uploads/sign` ‚Üí upload directly to Supabase Storage via signed URL ‚Üí store returned public URLs
- Submit: `POST /api/listings`
- On success: invalidate dashboard query + show success toast

### 8. Dashboard: my listings ‚Äî `frontend/src/components/dashboard/MyListings.tsx`
- Renders user's listings from dashboard data
- Status badge (active/pending/rejected)
- Edit + delete actions
- Edit opens a modal pre-filled with listing data (`PUT /api/listings/{id}`)
- Delete calls `DELETE /api/listings/{id}` after confirm dialog

### 9. Dashboard: liked properties ‚Äî `frontend/src/components/dashboard/LikedProperties.tsx`
- Grid of favorited listings from dashboard data
- Each card links to `/property/[id]`
- Unfavorite button

### 10. Dashboard: viewings ‚Äî `frontend/src/components/dashboard/MyViewings.tsx`
- List of upcoming viewings from dashboard data
- Status badge (pending/confirmed/cancelled)

### 11. Favorites toggle ‚Äî wired into `PropertyHero` and `SearchListingCard`
- Call `POST /api/listings/{id}/favorite`
- Optimistic update with TanStack Query mutation

### 12. API types ‚Äî `frontend/src/types/api.ts`
Add:
```typescript
export interface ApiListingBrief { id: string; title: string; location: string; price: number; currency: string; price_period: string | null; category: string; property_type: string; images: string[]; verified: boolean; is_new: boolean; status: string; bedrooms: number | null; bathrooms: number | null; size_sqm: number | null; floor_number: number | null; neighborhood: string | null; compound_name: string | null; views_count: number; created_at: string }
export interface ApiListingDetail extends ApiListingBrief { owner_id: string; agency_id: string | null; full_address: string | null; description: string | null; total_floors: number | null; neighborhood_id: string | null; amenities: string[]; latitude: number | null; longitude: number | null; lease_type: string | null; min_stay_months: number | null; available_date: string | null; payment_plan: unknown; delivery_date: string | null; title_deed_status: string | null; room_type: string | null; lifestyle_preferences: unknown; total_spots: number | null; filled_spots: number | null; availability: string | null; furnishing: string | null; utilities_included: boolean; bathroom_type: string | null; private_amenities: string[]; shared_amenities: string[]; housemates: unknown[]; similar_listings: ApiListingBrief[] }
```

### 13. UI types ‚Äî `frontend/src/types/index.ts`
Add mappers that convert `ApiListingBrief` ‚Üí `ListingBrief` (camelCase, UI-friendly).

### 14. Queries ‚Äî `frontend/src/lib/queries.ts`
- `useListings(filters)` ‚Äî `GET /api/listings` with filter params
- `useListing(id)` ‚Äî `GET /api/listings/{id}`
- `useFavorites()` ‚Äî `GET /api/listings/favorites`
- `useToggleFavorite()` ‚Äî mutation for `POST /api/listings/{id}/favorite`
- `useCreateListing()` ‚Äî mutation for `POST /api/listings`
- `useUpdateListing()` ‚Äî mutation for `PUT /api/listings/{id}`
- `useDeleteListing()` ‚Äî mutation for `DELETE /api/listings/{id}`
