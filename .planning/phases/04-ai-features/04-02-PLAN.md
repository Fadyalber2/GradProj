# Phase 4 — Plan 02: Frontend AI Features UI

## Goal
Wire all AI features into the frontend: ChatDrawer (SSE streaming), recommendations section, NL search bar, compatibility score on shared housing listings, and AI description generator in the add listing modal.

---

## Tasks

### 1. Chat drawer — `frontend/src/components/ai/ChatDrawer.tsx`
- Client component — sliding drawer from the right side of the screen
- State: `messages: {role: "user"|"assistant", content: string}[]`, `isOpen`, `isStreaming`
- Send message:
  - Append user message to list
  - Call `POST /api/ai/chat` with `{message, conversation_history}`
  - Handle SSE stream: read with `EventSource` or `fetch` + `ReadableStream`
  - Parse `data: {"token": "..."}` events → append tokens to last assistant message
  - `data: [DONE]` → stop streaming
  - If response is `{ai_unavailable: true}` → show "AI is offline" message
- UI: message bubbles (user right, assistant left), input bar, send button, loading dots while streaming
- "Clear chat" button
- Markdown rendering for assistant responses (use `react-markdown` or simple paragraph splits)

### 2. Floating AI button — `frontend/src/components/layout/FloatingAIButton.tsx`
- Fixed position bottom-right FAB
- Click → opens `ChatDrawer`
- Only rendered on non-admin, non-messages pages

### 3. Chatbot conditional — `frontend/src/components/layout/ChatbotConditional.tsx`
- Client component — renders `FloatingAIButton` + `ChatDrawer`
- Excluded on: `/admin/*`, `/messages` (has its own chat)
- Rendered globally in root layout

### 4. NL search bar — `frontend/src/components/find-homes/NLSearchBar.tsx`
- Prominent search input at top of Find Homes page
- Placeholder: "Describe your ideal property... (e.g. quiet 2-bed near Maadi under 8000 EGP)"
- On submit: `POST /api/ai/search` → replace listing grid with results
- Show `parsed_filters` as chip tags below input (e.g. "Location: Maadi", "Max: 8,000 EGP")
- If `ai_unavailable: true` → show "AI search unavailable, use filters" message
- Loading state with spinner

### 5. Recommendations section — `frontend/src/components/sections/RecommendationsSection.tsx`
- Shown on dashboard and homepage (for logged-in users)
- `GET /api/ai/recommendations`
- Renders 4–8 listing cards in a horizontal scroll
- "Based on your favorites" header (or "New listings" if no favorites)
- If `ai_unavailable: true` → show section with latest listings fallback

### 6. Compatibility score — `frontend/src/components/shared-housing/CompatibilityScore.tsx`
- Shown on shared housing property detail page
- For unauthenticated users: "Log in to see your compatibility score" CTA
- For authenticated users:
  - Pre-filled with user's `lifestyle_preferences` from profile if available
  - "Check Compatibility" button → `POST /api/ai/compatibility`
  - Shows score as circular progress (0–100) with color (green/yellow/red)
  - Shows reason chips below score
  - If `ai_unavailable: true` → "AI scoring temporarily unavailable"

### 7. AI description generator — inside `AddListingModal`
- "Generate Description" button in the description step
- Collects `title`, `property_type`, `category`, `city`, `bedrooms`, `bathrooms`, `size_sqm`, `amenities`, `price` from form state
- `POST /api/ai/description` → fills description textarea with English output
- Shows Arabic version in a separate read-only field
- Button disabled if Ollama unavailable (check via response)

### 8. Features section — `frontend/src/components/sections/FeaturesSection.tsx`
- Marketing section on homepage
- 3 feature cards: AI Chat, Fraud Detection, Roommate Matching
- Static content (no API calls)

### 9. Queries — additions to `frontend/src/lib/queries.ts`
- `useRecommendations()` — GET /api/ai/recommendations (enabled only when logged in)
- `useNLSearch()` — mutation for POST /api/ai/search
- `useCompatibility()` — mutation for POST /api/ai/compatibility
- `useGenerateDescription()` — mutation for POST /api/ai/description

### 10. API types — `frontend/src/types/api.ts`
Add:
```typescript
export interface ApiAISearchResponse {
  query: string
  parsed_filters: Record<string, unknown>
  results: ApiListingBrief[]
  total: number
}
export interface ApiCompatibilityResponse {
  listing_id: string
  compatibility_score: number
  reasons: string[]
}
export interface ApiDescriptionResponse {
  english: string
  arabic: string
}
export interface AiUnavailableResponse {
  ai_unavailable: true
}
```

### 11. SSE streaming utility — `frontend/src/lib/sse.ts`
- `streamChat(message, history, onToken, onDone, onError)` — handles the fetch + ReadableStream parsing
- Parses `data: {"token": "..."}` events
- Handles `data: [DONE]`
- Returns abort controller for cleanup
