# Phase 4 — Plan 01: Backend AI Features

## Goal
Implement all AI endpoints: natural language search, streaming chat (SSE), property recommendations (pgvector), roommate compatibility scoring, and bilingual description generation.

## Context
- All AI routes in `backend/app/ai/router.py`
- Ollama at `http://localhost:11434`, model `axiom-llm:latest`
- `{ai_unavailable: true}` returned (not a 5xx) when Ollama is down — frontend degrades gracefully
- pgvector `match_listings` RPC function must exist in Supabase DB

## Tasks

### 1. Ollama client — `backend/app/ai/ollama_client.py`
Already covered in Phase 2 Plan 01. Verify:
- `health()` → checks if `axiom-llm:latest` is loaded
- `generate(prompt, system)` → blocking text generation
- `generate_stream(prompt, system)` → async generator of token strings
- `embed(text)` → 768-dim float list

### 2. NL Search — `POST /api/ai/search`
- Check `ollama.health()` → return `AI_UNAVAILABLE` if down
- `_extract_filters_from_query(query)` — send query to Ollama with structured JSON extraction system prompt
  - Extract: `location`, `max_price`, `min_price`, `bedrooms`, `bathrooms`, `category`, `property_type`
  - Parse JSON from response (find first `{` to last `}`)
  - Fallback: return `{}` on any error
- Build Supabase query from extracted filters
  - Location: `city.ilike.%{location}% OR location.ilike.%{location}%`
  - All other fields: direct equality / gte / lte
- Return: `{query, parsed_filters, results: [ListingBrief], total}`

### 3. Chat SSE — `POST /api/ai/chat`
- Check `ollama.health()` → return `AI_UNAVAILABLE` if down
- System prompt: Egyptian real estate expert, bilingual (Arabic/English), focus on Cairo/Giza/Alexandria/New Capital/North Coast/Hurghada/Sharm
- Build `full_prompt` from last 6 messages of `conversation_history` + new `message`
- Return `StreamingResponse` with `media_type="text/event-stream"`
  - Each token: `data: {"token": "..."}\n\n`
  - End: `data: [DONE]\n\n`
  - Error: `data: {"error": "..."}\n\n`
- Headers: `Cache-Control: no-cache`, `Connection: keep-alive`, `X-Accel-Buffering: no`

### 4. Recommendations — `GET /api/ai/recommendations`
- Requires auth
- Fetch user's favorite listing IDs (up to 10)
- If no favorites: return 8 newest active listings
- If favorites exist: fetch first favorited listing's `category`, `city`, `embedding`
- **Primary path** (if embedding exists + Ollama healthy):
  - Call `supabase_admin.rpc("match_listings", {query_embedding, match_threshold: 0.5, match_count: 12, filter_category, filter_city})`
  - Return top 8 non-favorited matches
- **Fallback**: same category + city filter, order by `views_count DESC`, limit 8

### 5. Compatibility — `POST /api/ai/compatibility`
- Requires auth
- Body: `{listing_id, lifestyle_data: {...}}`
- Fetch listing → verify `category == "shared_housing"` (else 400)
- Get listing's `lifestyle_preferences`
- Send both preference objects to Ollama with scoring system prompt
- System prompt specifies JSON output: `{score: 0-100, reasons: [string]}`
- Clamp score to 0–100
- Fallback: score=50, reasons=[] on any Ollama error
- Return: `{listing_id, compatibility_score, reasons}`

### 6. Description generation — `POST /api/ai/description`
- Requires auth
- Body: `{title, property_type, category, city, bedrooms?, bathrooms?, size_sqm?, amenities?, price?, extra_notes?}`
- System prompt: professional Egyptian real estate copywriter, bilingual output JSON `{english, arabic}`
- Build detailed prompt from all fields
- Parse JSON response (first `{` to last `}`)
- Fallback: `{english: raw_text, arabic: ""}` if JSON parse fails
- Return `{english, arabic}`

### 7. pgvector RPC function (Supabase SQL migration)
Ensure this function exists in the DB — create migration if not present:
```sql
CREATE OR REPLACE FUNCTION match_listings(
  query_embedding vector(768),
  match_threshold float DEFAULT 0.5,
  match_count int DEFAULT 12,
  filter_category text DEFAULT NULL,
  filter_city text DEFAULT NULL
)
RETURNS TABLE (id uuid, similarity float)
LANGUAGE sql STABLE
AS $$
  SELECT id, 1 - (embedding <=> query_embedding) AS similarity
  FROM listings
  WHERE status = 'active'
    AND deleted_at IS NULL
    AND embedding IS NOT NULL
    AND (filter_category IS NULL OR category = filter_category)
    AND (filter_city IS NULL OR city ILIKE '%' || filter_city || '%')
    AND 1 - (embedding <=> query_embedding) > match_threshold
  ORDER BY embedding <=> query_embedding
  LIMIT match_count;
$$;
```
