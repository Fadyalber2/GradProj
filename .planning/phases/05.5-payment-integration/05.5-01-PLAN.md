**ENUMS** — wrap each in DO block to guard against duplicate creation:
```sql
DO $$ BEGIN
  CREATE TYPE payment_status AS ENUM ('pending', 'paid', 'failed', 'refunded');
EXCEPTION WHEN duplicate_object THEN NULL; END $$;

DO $$ BEGIN
  CREATE TYPE payment_provider AS ENUM ('paymob', 'fawry');
EXCEPTION WHEN duplicate_object THEN NULL; END $$;

DO $$ BEGIN
  CREATE TYPE payment_type AS ENUM ('deal_commission', 'agency_subscription');
EXCEPTION WHEN duplicate_object THEN NULL; END $$;

DO $$ BEGIN
  CREATE TYPE deal_type AS ENUM ('sale', 'rental');
EXCEPTION WHEN duplicate_object THEN NULL; END $$;

DO $$ BEGIN
  CREATE TYPE deal_status AS ENUM ('pending', 'invoiced', 'commission_paid', 'disputed');
EXCEPTION WHEN duplicate_object THEN NULL; END $$;

DO $$ BEGIN
  CREATE TYPE subscription_status AS ENUM ('active', 'expired', 'cancelled');
EXCEPTION WHEN duplicate_object THEN NULL; END $$;

DO $$ BEGIN
  CREATE TYPE subscription_plan AS ENUM ('none', 'starter', 'pro', 'enterprise');
EXCEPTION WHEN duplicate_object THEN NULL; END $$;
```

**Table: `deals`** — create before `payments` because payments has FK to deals:
- `id uuid PRIMARY KEY DEFAULT gen_random_uuid()`
- `listing_id uuid NOT NULL REFERENCES listings(id) ON DELETE CASCADE`
- `owner_id uuid NOT NULL REFERENCES profiles(id) ON DELETE CASCADE`
- `deal_type deal_type NOT NULL`
- `deal_value integer NOT NULL` (piasters — EGP * 100)
- `commission_pct numeric(5,2) NOT NULL` (e.g., 2.00 or 1.50)
- `commission_amount integer NOT NULL` (piasters)
- `status deal_status NOT NULL DEFAULT 'pending'`
- `created_at timestamptz DEFAULT now()`
- `UNIQUE (listing_id, owner_id)` — CRITICAL: prevents duplicate deal reports per Pitfall 5 in research
- `CREATE INDEX IF NOT EXISTS idx_deals_owner ON deals (owner_id)`
- `CREATE INDEX IF NOT EXISTS idx_deals_listing ON deals (listing_id)`
- RLS: `ALTER TABLE deals ENABLE ROW LEVEL SECURITY`
- Policy: `CREATE POLICY deals_owner_read ON deals FOR SELECT USING (owner_id = auth.uid())`

**Table: `agency_subscriptions`** — create before `payments` because payments has FK:
- `id uuid PRIMARY KEY DEFAULT gen_random_uuid()`
- `agency_id uuid NOT NULL REFERENCES agencies(id) ON DELETE CASCADE`
- `plan subscription_plan NOT NULL`
- `price integer NOT NULL` (piasters per month)
- `starts_at timestamptz NOT NULL DEFAULT now()`
- `ends_at timestamptz NOT NULL` (set by service layer to starts_at + INTERVAL '30 days')
- `status subscription_status NOT NULL DEFAULT 'active'`
- `created_at timestamptz DEFAULT now()`
- `UNIQUE (agency_id, starts_at)`
- RLS: `ALTER TABLE agency_subscriptions ENABLE ROW LEVEL SECURITY`
- Policy: `CREATE POLICY subscriptions_owner_read ON agency_subscriptions FOR SELECT USING (agency_id IN (SELECT id FROM agencies WHERE owner_id = auth.uid()))`

**Table: `payments`** — references both deals and agency_subscriptions:
- `id uuid PRIMARY KEY DEFAULT gen_random_uuid()`
- `user_id uuid NOT NULL REFERENCES profiles(id) ON DELETE CASCADE`
- `payment_type payment_type NOT NULL`
- `amount integer NOT NULL` (piasters — EGP * 100)
- `currency text NOT NULL DEFAULT 'EGP'`
- `provider payment_provider NOT NULL`
- `payment_method text` (nullable — 'card', 'wallet', or 'fawry')
- `status payment_status NOT NULL DEFAULT 'pending'`
- `merchant_ref uuid UNIQUE NOT NULL DEFAULT gen_random_uuid()` — CRITICAL: UNIQUE enforces webhook idempotency; duplicate webhook delivery hits this and returns 200 without double-processing
- `provider_ref text` (nullable — Paymob order_id or Fawry referenceNumber from gateway response)
- `fawry_ref text` (nullable — 10-digit cash reference code displayed to user at Fawry POS)
- `deal_id uuid REFERENCES deals(id) ON DELETE SET NULL`
- `subscription_id uuid REFERENCES agency_subscriptions(id) ON DELETE SET NULL`
- `created_at timestamptz DEFAULT now()`
- `paid_at timestamptz` (nullable — set when status transitions to 'paid')
- `CREATE INDEX IF NOT EXISTS idx_payments_user ON payments (user_id)`
- `CREATE INDEX IF NOT EXISTS idx_payments_merchant ON payments (merchant_ref)`
- RLS: `ALTER TABLE payments ENABLE ROW LEVEL SECURITY`
- Policy: `CREATE POLICY payments_owner_read ON payments FOR SELECT USING (user_id = auth.uid())`
- NO write policy for authenticated users — all writes via FastAPI service role key (intentional security boundary)

**ALTER TABLE agencies** — add denormalized subscription state for fast reads:
```sql
ALTER TABLE agencies
  ADD COLUMN IF NOT EXISTS subscription_plan subscription_plan NOT NULL DEFAULT 'none',
  ADD COLUMN IF NOT EXISTS subscription_expires_at timestamptz,
  ADD COLUMN IF NOT EXISTS listing_quota integer NOT NULL DEFAULT 5;
```
Rationale comment in SQL: these 3 fields are denormalized for fast agency list queries. Agency subscription records in agency_subscriptions are the canonical audit trail.
