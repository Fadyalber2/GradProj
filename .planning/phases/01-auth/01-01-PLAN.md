## Project Context

- Backend: FastAPI + Python at `G:/AI/AXIOM-V2/backend/`
- Database: Supabase (project `pgaqqseqwtgsuihbswnv`, eu-west-1) with 13 tables including `profiles`
- Auth: Supabase issues ES256 JWTs; backend verifies via JWKS endpoint
- Two Supabase clients: `supabase_client` (anon key, user-context) and `supabase_admin` (service role, bypasses RLS)
- DB trigger `on_auth_user_created` auto-creates a `profiles` row when a Supabase auth user is created
- Admin auth is entirely separate from Supabase — HS256 JWT, credentials from env vars
- Phone OTP via Twilio Verify v2 — credentials in `.env` (optional, gracefully disabled if missing)
- Tests mock all Supabase calls — no real DB connections in test suite

## Key Invariants

- `UserRole = "user" | "admin"` — no broker, no seeker
- `owner_id` / `ownerId` everywhere — never `broker_id`
- All auth endpoints under `/api/auth/`; admin auth under `/api/admin/auth/`
- Router functions are thin — validate input, call service/Supabase, return response
- Never use `print()` — use Python `logging` module

## Supabase Admin API (Python SDK)

```python
# Create user — admin client required
auth_resp = supabase_admin.auth.admin.create_user({
    "email": "user@example.com",
    "password": "securepass",
    "user_metadata": {"full_name": "John Doe"},
    "email_confirm": True,  # skip email confirmation
})
user_id = auth_resp.user.id

# Update profiles table
supabase_admin.table("profiles").update({"phone": "+201234567890"}).eq("id", user_id).execute()

# Fetch profile
result = supabase_admin.table("profiles").select("*").eq("id", user_id).single().execute()
profile = result.data
```

## Twilio Verify v2

```python
from twilio.rest import Client
client = Client(settings.twilio_account_sid, settings.twilio_auth_token)

# Send OTP
verification = client.verify.v2.services(settings.twilio_verify_service_sid).verifications.create(
    to="+201234567890", channel="sms"
)
# verification.status == "pending"

# Check OTP
check = client.verify.v2.services(settings.twilio_verify_service_sid).verification_checks.create(
    to="+201234567890", code="123456"
)
# check.status == "approved" means success
```

## JWT (PyJWT)

```python
import jwt
from jwt.algorithms import ECAlgorithm

# Decode ES256
payload = jwt.decode(token, public_key, algorithms=["ES256"], audience="authenticated")

# Decode HS256
payload = jwt.decode(token, settings.jwt_secret, algorithms=["HS256"], audience="authenticated")

# Encode (admin tokens only)
token = jwt.encode({"sub": username, "role": "admin", "exp": exp}, settings.jwt_secret, algorithm="HS256")
```
