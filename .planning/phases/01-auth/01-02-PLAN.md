# Phase 1 — Plan 02: Frontend Auth Wiring

## Goal
Wire the frontend auth flow end-to-end: Zustand store, API client, login/signup/forgot-password pages, OAuth (Google + Facebook), and 401 auto-logout.

## Context
- Frontend: `G:\AI\AXIOM-V2\frontend\`
- Auth is Supabase JS client-side + FastAPI profile fetching
- JWT is auto-injected by `api.ts` on every API call

---

## Tasks

### 1. Supabase JS client — `frontend/src/lib/supabase.ts`
- Export singleton `supabase` created with `createBrowserClient(NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY)`
- Use `@supabase/ssr` package

### 2. API client — `frontend/src/lib/api.ts`
- Base URL from `process.env.NEXT_PUBLIC_API_URL ?? "http://localhost:8000"`
- `api.get / post / put / patch / delete` — all auto-inject `Authorization: Bearer <token>` from Zustand store
- On 401 response: call `useAuthStore.getState().logout()` then redirect to `/login`
- Export `ApiError` class with `status`, `statusText`, `body`
- Export `serverFetch(path, options)` — for SSR use (no auth injection, 8s timeout)

### 3. Auth store — `frontend/src/stores/authStore.ts`
Zustand store with:
- State: `user: AuthUser | null`, `session: Session | null`, `isLoading: boolean`
- `initialize()` — calls `supabase.auth.getUser()`, fetches `/api/auth/me` on success. Retries up to 3× on 401 (profile trigger lag).
- `signup(email, password, fullName, phone?, countryCode?, gender?)` — calls `POST /api/auth/signup` then `signInWithPassword` then `fetchProfile`
- `login(email, password)` — calls `supabase.auth.signInWithPassword` then `fetchProfile`
- `loginWithGoogle()` — `supabase.auth.signInWithOAuth({ provider: "google" })`
- `loginWithFacebook()` — `supabase.auth.signInWithOAuth({ provider: "facebook" })`
- `logout()` — calls `supabase.auth.signOut()`, clears store
- `fetchProfile()` — `GET /api/auth/me` → updates `user` in store
- `refreshProfile()` — re-fetches profile (used after profile updates)

### 4. Auth types — `frontend/src/types/index.ts`
Add:
```typescript
export interface AuthUser {
  id: string
  email: string
  fullName: string | null
  avatarUrl: string | null

  bio: string | null
  role: "user" | "admin"
  isVerifiedSeller: boolean
  gender: string | null
  countryCode: string | null
  badges: string[]
  age: number | null
  occupation: string | null
  lifestylePreferences: Record<string, unknown> | null
  createdAt: string | null
}
```

### 5. Auth callback — `frontend/src/app/auth/callback/page.tsx`
- Handle OAuth redirect: call `supabase.auth.exchangeCodeForSession(code)` → redirect to `/dashboard`
- Show loading spinner while processing

### 6. Login page — `frontend/src/app/(auth)/login/page.tsx` + `frontend/src/components/auth/LoginForm.tsx`
- Email + password fields with react-hook-form + zod validation
- Submit → `store.login()` → redirect to `/dashboard`
- Show error toast on failure
- Link to `/signup` and `/forgot-password`
- OAuth buttons (Google, Facebook) via `OAuthButton` component

### 7. Signup page — `frontend/src/app/(auth)/signup/page.tsx` + `frontend/src/components/auth/SignUpForm.tsx`
- Fields: full name, email, password, phone (optional)
- Phone OTP flow: if phone provided, send OTP via `POST /api/auth/send-phone-otp`, show 6-digit input, verify via `POST /api/auth/verify-phone-otp`
- Submit → `store.signup()` → redirect to `/dashboard`
- OAuth buttons
- `PhoneOTPInput` component in `frontend/src/components/auth/PhoneOTPInput.tsx`

### 8. Forgot password — `frontend/src/app/(auth)/forgot-password/page.tsx` + `frontend/src/components/auth/ForgotPasswordForm.tsx`
- Email input → call `supabase.auth.resetPasswordForEmail(email, { redirectTo: "..." })`
- Show success message

### 9. Reset password — `frontend/src/app/(auth)/reset-password/page.tsx` + `frontend/src/components/auth/ResetPasswordForm.tsx`
- Read token from URL hash
- New password input → `supabase.auth.updateUser({ password })`
- Redirect to `/login` on success

### 10. OAuth buttons — `frontend/src/components/auth/OAuthButton.tsx`
- Renders a styled button for a provider (Google or Facebook)
- `OAuthIcons.tsx` — SVG icons for Google and Facebook

### 11. Route middleware — `frontend/middleware.ts`
- Protected routes (`/dashboard`, `/messages`): redirect to `/login` if no `sb-*-auth-token` cookie
- Auth-only routes (`/login`, `/signup`, `/forgot-password`): redirect to `/dashboard` if already logged in
- Use `@supabase/ssr` `createServerClient` to read session from cookie

### 12. App providers — `frontend/src/providers/Providers.tsx`
- Wrap app in `QueryClientProvider`
- `AuthInitializer` component that calls `store.initialize()` on mount
- Wrap in `Toaster` (shadcn/ui)

### 13. Root layout — `frontend/src/app/layout.tsx`
- Dark mode class on `<html>`
- Wrap `{children}` in `<Providers>`
- Import globals.css
