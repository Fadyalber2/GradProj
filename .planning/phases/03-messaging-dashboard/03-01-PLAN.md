  <name>Task 2: Messages router</name>
  <files>backend/app/messages/router.py</files>
  <action>
  Create `backend/app/messages/router.py`. This is the most complex router in Phase 3.

  **Helper functions (private):**
  - `_get_other_user_id(conv, user_id)` — returns the other participant's ID from `user_a_id`/`user_b_id`
  - `_verify_participation(conv, user_id)` — raises 403 if user is not `user_a_id` or `user_b_id`
  - `_fetch_conversation(conversation_id)` — fetches by ID, raises 404 if not found; selects `id, user_a_id, user_b_id, status, initiated_by`

  **Route ordering (critical — declare in this order):**

  1. `POST /block` — block a user
     - Check self-block (400)
     - Check `blocked_users` table for existing block → return early if already blocked
     - Insert into `blocked_users` with `blocker_id`, `blocked_id`, optional `reason`
     - Best-effort: find all `pending`/`accepted` conversations between the two users, set `status='rejected'`
     - Return `{"detail": "User blocked"}`

  2. `DELETE /block/{blocked_user_id}` — unblock
     - Delete from `blocked_users` where `blocker_id=user_id AND blocked_id=blocked_user_id`
     - 404 if no record found

  3. `GET /blocked` — list blocked users with profile enrichment
     - Select from `blocked_users` where `blocker_id=user_id`, ordered by `created_at DESC`
     - For each, fetch `full_name` and `avatar_url` from `profiles`
     - Return `list[BlockedUserResponse]`

  4. `GET /conversations` — list conversations
     - Call `supabase_admin.rpc("get_user_conversations", {"p_user_id": user_id})`
     - Fetch `status` and `initiated_by` for all returned IDs via single `.in_()` query on `conversations` table
     - Skip conversations with `status='rejected'`
     - Enrich each with other user's profile (`full_name`, `avatar_url`)
     - Return `list[ConversationResponse]`

  5. `POST /conversations` — create or get conversation
     - Self-check (400)
     - Bidirectional block check: `.or_("and(blocker_id.eq.X,blocked_id.eq.Y),and(blocker_id.eq.Y,blocked_id.eq.X)")` on `blocked_users`, limit 1 → 403 if found
     - Check existing conversation: `.or_()` both directions, optionally filter by `listing_id`
     - If existing and was `rejected`: reset to `pending`, update `initiated_by`, send `message_request` notification
     - If new: insert with `status='pending'`, `initiated_by=user_id`; send `message_request` notification to `other_user_id`
     - If `initial_message` provided: insert into `messages`
     - Return `{"id": conversation_id}`

  6. `POST /conversations/{conversation_id}/accept`
     - `_fetch_conversation` + `_verify_participation`
     - If `initiated_by == user_id` → 403 ("Only the recipient can accept")
     - If already accepted → return early
     - If rejected → 400 (cannot accept rejected)
     - Update `status='accepted'`
     - Notify initiator: type `message_request_accepted`

  7. `POST /conversations/{conversation_id}/reject`
     - `_fetch_conversation` + `_verify_participation`
     - If `initiated_by == user_id` → 403 ("Only the recipient can reject")
     - If already rejected → return early
     - Update `status='rejected'`

  8. `GET /conversations/{conversation_id}` — get messages
     - `_fetch_conversation` + `_verify_participation`
     - Select all messages ordered by `created_at ASC`
     - Return `list[MessageResponse]`

  9. `POST /conversations/{conversation_id}` — send message
     - `_fetch_conversation` + `_verify_participation`
     - If `status='rejected'` → 403
     - If `status='pending'` and `initiated_by == user_id` → 403 ("Waiting for acceptance")
     - If `status='pending'` and recipient is sending → auto-accept + notify initiator
     - Insert message with `text`, optional `attachment_url/name/size`
     - Insert `new_message` notification for the other user
     - Return `MessageResponse`

  All DB operations wrapped in try/except raising HTTPException. Use `supabase_admin` throughout.
