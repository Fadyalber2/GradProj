# Phase 3 ‚Äî Plan 02: Frontend Messaging, Dashboard & Notifications UI

## Goal
Build the Messages page (inbox + chat), Dashboard page, Notification Bell (live via Supabase Realtime), and all associated components.

---

## Tasks

### 1. Dashboard page ‚Äî `frontend/src/app/dashboard/page.tsx`
- Protected client component (or server component with client children)
- On mount: call `GET /api/dashboard/me` via TanStack Query (gated on `!!session`)
- Layout: sidebar profile + stat cards row + listings grid + recent messages + liked properties + viewings

### 2. Dashboard: profile card ‚Äî `frontend/src/components/dashboard/DashboardProfile.tsx`
- Shows avatar, name, verified badge, bio, phone
- "Edit Profile" button ‚Üí opens edit modal (PUT /api/auth/me)

### 3. Dashboard: stats ‚Äî `frontend/src/components/dashboard/DashboardStats.tsx`
- 4 stat cards from `analytics` array: label, value, trend arrow + percent
- Framer Motion entrance animation

### 4. Dashboard: recent messages ‚Äî `frontend/src/components/dashboard/RecentMessages.tsx`
- Last 5 conversations preview
- Each row: avatar, name, last message excerpt, time
- Clicking navigates to `/messages?conversation=<id>`

### 5. Messages page ‚Äî `frontend/src/app/messages/page.tsx`
- Split view: `InboxSidebar` (left) + `ChatArea` (right)
- URL param `?conversation=<id>` selects active conversation
- On mobile: full-width, toggle between sidebar and chat

### 6. Inbox sidebar ‚Äî `frontend/src/components/messages/InboxSidebar.tsx`
- List of conversations from `GET /api/messages/conversations`
- Each row: avatar, name, last message preview, timestamp, unread badge
- Search input to filter conversations client-side
- Message requests section (status=pending) separated from active conversations
- `RequestBar` component for accept/reject

### 7. Chat area ‚Äî `frontend/src/components/messages/ChatArea.tsx`
- Fetch messages from `GET /api/messages/conversations/{id}`
- Render message bubbles with sent/received alignment
- `DateSeparator` component between messages from different days
- `ScrollToBottom` FAB ‚Äî appears when user scrolls up, jumps to latest
- Send message: text input + submit ‚Üí `POST /api/messages/conversations/{id}`
- Supabase Realtime subscription: listen for new messages in this conversation ‚Üí append to list
- `ConversationMenu` (‚ãÆ button) ‚Üí block / report options

### 8. Message request bar ‚Äî `frontend/src/components/messages/RequestBar.tsx`
- Shows for conversations with `status = "pending"` and current user is recipient
- "Accept" ‚Üí `PUT /api/messages/conversations/{id}/accept`
- "Reject" ‚Üí `PUT /api/messages/conversations/{id}/reject`

### 9. Conversation menu ‚Äî `frontend/src/components/messages/ConversationMenu.tsx`
- Dropdown with: Block User, Report Conversation
- Block ‚Üí `PUT /api/messages/conversations/{id}/block`
- Report ‚Üí opens reason modal ‚Üí `POST /api/messages/conversations/{id}/report`

### 10. Messages skeleton ‚Äî `frontend/src/components/messages/MessagesSkeleton.tsx`
- Loading skeleton for inbox sidebar and chat area

### 11. Messages empty state ‚Äî `frontend/src/components/messages/EmptyState.tsx`
- Shown when inbox is empty or no conversation selected
- Friendly illustration + CTA

### 12. Date separator ‚Äî `frontend/src/components/messages/DateSeparator.tsx`
- Renders a horizontal line with date label (e.g. "Yesterday", "March 15") between message groups

### 13. Scroll-to-bottom FAB ‚Äî `frontend/src/components/messages/ScrollToBottom.tsx`
- Floating action button that appears when chat is scrolled up
- Clicking scrolls chat area to bottom
- Shows unread count badge

### 14. Notification bell ‚Äî `frontend/src/components/layout/NotificationBell.tsx`
- In `Navbar`
- Fetches `GET /api/notifications` on mount
- Shows unread count badge
- Dropdown: list of recent notifications, each with title + body + time
- "Mark all read" button ‚Üí `PUT /api/notifications/read-all`
- Single click ‚Üí `PUT /api/notifications/{id}/read`
- **Supabase Realtime**: subscribe to `notifications` table filtered by `user_id = currentUser.id` ‚Üí increment badge count on new row

### 15. Navbar ‚Äî `frontend/src/components/layout/Navbar.tsx`
- Logo, nav links (Find Homes, Blog, Agencies)
- Right side: NotificationBell (if logged in), Dashboard link, Login/Signup buttons
- Mobile: hamburger menu

### 16. Queries ‚Äî additions to `frontend/src/lib/queries.ts`
- `useConversations()` ‚Äî GET /api/messages/conversations
- `useMessages(conversationId)` ‚Äî GET /api/messages/conversations/{id}
- `useSendMessage(conversationId)` ‚Äî mutation
- `useStartConversation()` ‚Äî mutation
- `useAcceptConversation()`, `useRejectConversation()`, `useBlockConversation()`
- `useNotifications()` ‚Äî GET /api/notifications
- `useMarkAllRead()` ‚Äî mutation
- `useDashboard()` ‚Äî GET /api/dashboard/me

### 17. API types ‚Äî `frontend/src/types/api.ts`
Add `ApiConversation`, `ApiMessage`, `ApiNotification`, `ApiDashboardResponse`, `ApiDashboardListing`, `ApiDashboardMessage`, `LikedPropertyBrief`, `ApiViewingBrief`.
